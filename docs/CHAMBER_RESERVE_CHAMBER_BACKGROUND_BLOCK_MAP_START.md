# ChamberReserve Chamber Background Block Map Start

作成日: 2026-06-22

## 目的

採用した開口状態のチャンバー正面模式図を、4 x 3 ブロック選択UIの背景として組み込む。

背景画像自体には選択枠を焼き込まない。React 側で予約状態を持つ 4 x 3 の操作レイヤーを重ね、利用者が「棚のどの位置を使うか」を直感的に判断できる状態にする。

## 背景

Phase 3.5 で検索画面は「候補比較 + 4 x 3 ブロック選択」へ再構成された。現在の `BlockGrid` は予約ルール上は正しく動くが、見た目はまだ抽象的なグリッドであり、実チャンバー庫内の上段/中段/下段との対応が弱い。

今回採用する背景は、次の特徴を持つ。

- 扉は表示しない。
- ロゴや `AR SERIES` などの文字は表示しない。
- 棚が2枚見える開口状態の正面模式図。
- 背景自体には4 x 3の選択枠を持たない。
- 4 x 3 操作レイヤーは画像上にオーバーレイする。

## 長期計画上の位置づけ

- Phase 3.5 の後続UXリファインとして扱う。
- Phase 4 の固定条件型チャンバー対応へ進む前に、ブロック選択の視覚基準を固める。
- サーバーAPI、SQLite、予約判定ロジックを増やすフェーズではない。

## 採用アセット

採用する背景は次のファイルとする。

- `src/assets/espec-ar-front-background.svg`

画像は 960 x 720 viewBox を基準に作られている。4 x 3 操作レイヤーの基準領域は次の通り。

```text
x=280 y=126 width=540 height=405
```

CSSでは、親要素を `aspect-ratio: 4 / 3` にしたうえで次の比率を使う。

```text
left: 29.1667%
top: 17.5%
width: 56.25%
height: 56.25%
```

この領域は 540 x 405 なので、4列 x 3行に分割すると各ブロックは 135 x 135 の正方形になる。

## 実装方針

### 1. 背景と操作レイヤーを分離する

背景SVGは視覚コンテキストだけを担う。選択可能状態、予約済み、一時利用停止、候補ブロック、影響予約などの業務状態はすべてReactのオーバーレイボタンで表現する。

これにより、将来チャンバーごとに背景画像や庫内寸法が変わっても、画像とオーバーレイ座標を差し替えるだけで対応できる。

### 2. `BlockGrid` の責務を保つ

既存の `BlockGrid` は、選択状態、候補状態、予約済み、一時停止、ドラッグ選択、readonly 表示をすでに持っている。今回もこの責務を維持する。

推奨実装は次のいずれかとする。

- `BlockGrid` に `surface?: 'plain' | 'chamber'` を追加し、既定値を `plain` にする。
- または `ChamberFrontBlockMap` を新規作成し、`BlockGrid` から大きい表示だけを委譲する。

既存呼び出しを壊さないため、compact 候補ミニマップは原則 `plain` のままにする。背景付き表示は、利用者が実際に選択・確認する大きなパネルを中心に適用する。

### 3. 大きい選択面にだけ背景を使う

背景付き表示を優先する場所:

- 検索条件側の利用ブロック選択。
- 選択中候補の詳細パネル。
- 予約ボードの大きい利用状況表示。
- 管理者一時利用停止のブロック選択。

背景なし表示を維持してよい場所:

- 候補行内の小さいミニマップ。
- 予約一覧行の小さいブロック表示。
- モバイル幅で背景が小さくなりすぎる箇所。

### 4. オーバーレイセルは固定寸法で揺らさない

オーバーレイは `position: absolute` の 4 x 3 CSS Grid とする。

- `grid-template-columns: repeat(4, minmax(0, 1fr))`
- `grid-template-rows: repeat(3, minmax(0, 1fr))`
- `gap: 0`
- 各セルは `button`
- `box-sizing: border-box`
- ラベル、hover、focus、状態表示でセル寸法が変わらないこと

通常セルは背景をほぼ透明にする。状態を持つセルだけ半透明色、太枠、斜線などを使う。

### 5. 状態色とパターン

色は既存の Phase 3.5 方針を維持する。

- selected: teal の半透明塗り + 太枠。
- candidate: teal 系の薄い塗りまたは枠。
- reserved: gray の半透明塗り。
- suspended: red 系の半透明塗り + 斜線。
- impact: amber 系の半透明塗り。
- disabled: 低コントラスト + 操作不可。
- empty: 透明に近い背景 + hover時のみ薄い強調。

色だけに依存しないよう、一時停止や影響予約にはパターンやアイコンに近い視覚差を加える。ただしセル内テキストを増やしすぎない。

### 6. ラベルとアクセシビリティ

背景が視覚的に棚を示すため、セル内の文字は控えめにする。

- `aria-label` には既存の `block.label` を使う。
- `aria-pressed` を維持する。
- キーボードフォーカスは見えるようにする。
- 表示ラベルは小さく右上または左上に置くか、詳細パネル外の行/列ラベルに分離する。
- ユーザーが `左1/左2/右2/右1` と `上段/中段/下段` を把握できる最低限のラベルは維持する。

### 7. モバイル対応

モバイル幅では背景付きブロックマップを横幅いっぱいに置き、候補比較リストの下または上に流す。

必要条件:

- 横スクロールを発生させない。
- セルが小さくなりすぎる場合でもタップ可能サイズを守る。
- 背景が視認性を邪魔する場合は、モバイルだけ背景の不透明度を下げる。

### 8. ブラウザ検証を更新する

既存の Phase 3.5 ブラウザ検証に加え、背景付き4 x 3の観点を明示する。

追加で確認すること:

- 背景画像が読み込まれている。
- 4 x 3 枠が庫内領域内に収まっている。
- 棚板と行境界が大きくずれていない。
- 右端の枠や棚がはみ出していない。
- 選択、予約済み、一時停止、候補状態が背景上で読める。
- デスクトップとモバイルで文字やボタンが重ならない。

検証スクリプトを追加する場合は、`tmp_validation/chamber_background_block_map_check.cjs` のような専用名にする。

## Scope

- 採用SVG背景をアプリUIで使う。
- `BlockGrid` または新規コンポーネントで背景付き 4 x 3 オーバーレイを実装する。
- 大きいブロック選択/詳細/ボード/一時停止表示へ背景付き表示を適用する。
- 候補ミニマップは必要に応じて plain 表示のまま維持する。
- 状態色、hover、focus、readonly、drag selection を維持する。
- ブラウザ確認スクリプトとスクリーンショットを更新する。
- 実装サマリーを追加または既存サマリーに追記する。

## Out of Scope

- 予約ロジック変更。
- API変更。
- SQLiteスキーマ変更。
- 新しいチャンバー種別の追加。
- 複数チャンバー登録CRUD。
- 奥行き、高さ、重量、通風クリアランス管理。
- アカウント管理。
- 仮押さえ。
- 高度なPINセキュリティ。
- 通知連携。
- 外部画像生成や新規デザイン依存パッケージの追加。

## 守る仕様

- 最初の画面は条件検索。
- 4 x 3 はチャンバー全体の正面投影。
- 背景画像に選択枠を焼き込まない。
- 断片化したブロック割り当ては候補に出さない。
- 温度サイクル型の 25°C 搬入、運転期間、25°C 搬出を弱めない。
- 4桁PIN編集/削除、利用後削除不可、管理者一時利用停止の影響予約表示を維持する。
- 既存の未コミット変更を戻さない。

## 実現されるユーザビリティ

- 利用者は抽象的なマスではなく、実際の庫内の棚位置としてブロックを理解できる。
- `上段/中段/下段` と実チャンバー内の棚の対応が見え、位置指定予約の誤解が減る。
- サイズだけ予約と位置まで指定する予約の違いが直感的になる。
- 予約済み、一時停止、影響予約を同じ庫内図上で比較でき、現場での配置判断が速くなる。
- 背景に枠を焼き込まないため、状態色や予約ルールの変更に強い。
